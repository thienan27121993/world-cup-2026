### 1. プロダクトの目的
2026 FIFAワールドカップ（2026/6/11〜7/19、米国・カナダ・メキシコ共催、48チーム・全104試合）を、**日本のユーザー向け・日本時間(JST)基準**で快適に追えるモバイルアプリ。コア要件は次の4つ＋拡張：
1. 日程確認（JST、日付ごと）
2. 試合前のプッシュ通知
3. 注目試合の色分け（特に日本代表）
4. 中継リンク（日本国内の配信）
5. 拡張：次の試合の自動繰り上げ／ライブスコア・順位表のリアルタイム連携

### 2. 既存の成果物（このフォルダにある）
- `index.html` — 単一ファイルのPWA本体（バニラJS、約42KB）。タブ＝今日/日程/順位/設定。
- `sw.js` — Service Worker（オフラインキャッシュ／予約通知／本物のWeb Push受け口）。
- `manifest.webmanifest`, `icon.svg` — インストール用。
- `起動_Mac.command` / `起動_Windows.bat` / `起動_Linux.sh` — ローカル起動スクリプト（`python -m http.server 8000`）。
- `README.md` — ホスティング・通知・ライブ連携プロキシの手順。

### 3. 実装済みの機能と設計
- **デザイン**: 落ち着いた「インク+砂金」配色。必ず以下のCSS変数トークンを踏襲すること（彩度を上げない）。
  `--bg:#0e1116; --panel:#161a22; --txt:#e6e8ee; --muted:#7d8597; --sand:#c9a96a(アクセント); --jp:#c8203f(日本/赤); --feat:#b8924a(注目/黄土); --knock:#6fae9b(決勝T/セージ); --live:#d9534f`。見出しは Fraunces(serif)、数字は DM Mono、本文は Zen Kaku Gothic New。
- **色分け**: 日本代表=赤、注目試合=黄土、決勝T=セージ、LIVE中=赤枠。
- **トップ(ヒーロー)**: 30秒ごとに再評価し、次の試合へ自動繰り上げ。開催中の試合があれば自動でLIVE表示（スコア・経過分）に切替。試合前はカウントダウン。優先順位は「LIVE中の試合 > 次の日本戦 > 直近の試合」。
- **通知（3段階）**: ①アプリ起動中の `setTimeout`、②`Notification Triggers`（`registration.showNotification` に `showTrigger: new TimestampTrigger(t)`、アプリを閉じても発火・Chromium系）、③本物のWeb Push（`sw.js` の `push` イベントで受信、VAPID＋配信サーバーが必要＝未実装）。設定で「日本戦を自動ON」「注目試合を自動ON」「通知タイミング(10/15/30/60/120分前)」。
- **中継**: 試合ごとのシートから DAZN(全試合/日本戦無料)・NHKプラス・TVer(日テレ/フジ無料) を開く。
- **順位表**: 結果からクライアント側で自動計算（勝点→得失点差→総得点でソート、上位2を突破圏として✅、日本をハイライト）。「デモ表示」で開幕前でも体験可能。
- **永続化**: `localStorage`（通知登録・設定・お気に入りチーム）。

### 4. データモデル（重要・踏襲すること）
- 試合: `MATCHES[]`、各要素 `{ id, iso(JST,例"2026-06-15T05:00:00+09:00"), grp("A"〜"L"/"R32"/"R16"/"QF"/"SF"/"3位"/"決勝"), h, a, city, fh(国旗絵文字), fa, feat:bool, jp:bool, knock:bool, tbd:bool(時刻未定) }`。
- 結果オーバーレイ: `RESULTS = { [matchId]: { sh, sa, status:"NS"|"LIVE"|"FT", minute } }`。順位表はこれから計算。
- **ライブ連携の契約**: アプリは `GET {apiBase}/live` を叩き、`{ updated, results: { "<matchId>": {sh,sa,status,minute} } }` を期待。外部スポーツAPIはCORS／キー漏洩のため**必ず自前プロキシ経由**（READMEにCloudflare Workerの実装例あり）。

### 5. 確定済みの正確な事実（捏造しないこと）
- ET→JST は **+13時間**。
- 日本はグループF（オランダ・スウェーデン・チュニジア）。日本戦(JST)：6/15 05:00 vsオランダ、6/21 13:00 vsチュニジア、6/26 08:00 vsスウェーデン。
- 決勝：7/20 04:00 JST、NY/NJ メットライフ。
- 日本の放映：DAZN(全104試合・日本戦無料)、NHK(地上波一部)、日テレ・フジ(一部＋TVer無料)。
- **不確定**：グループ第2・3節の一部時刻、決勝トーナメントの組合せ（進行で確定）。現状データは2026/6/8時点。第1節は確定、以降は確定分のみ。憶測で全104試合を埋めず、不明分は API 取得かプレースホルダにすること。

### 7. ロードマップ（優先度順の候補）
1. **HTTPS公開**（Netlify/Cloudflare Pages）→ スマホインストール＆通知の有効化。
2. **Web Push 最小バックエンド**：VAPID鍵生成、購読(subscription)保存、各試合キックオフ前に `web-push` で送信するスケジューラ。
3. **ライブ連携プロキシ**：football-data.org か API-Football をラップする Cloudflare Worker。`/live` 契約に整形、`ID_MAP` を整備。
4. **全104試合データの整備**（できればAPIから取得して生成）。
5. **決勝Tブラケット**表示と**3位進出ロジック**。
6. （任意）UIを Vite + React/TypeScript へ移行し、保守性・テストを強化。

### 8. 最初にやること
1. このフォルダの既存ファイルを読み、現状を要約して私に確認を取る。
2. プロジェクト構成を提案する。デフォルトは「既存PWAを壊さず、`/server`(Web Push)と`/proxy`(ライブ)を足す段階的拡張」。フレームワーク移行を私が望む場合のみ大改修する。`.gitignore` と最小限の `package.json` を用意し、git 初期化を提案。
3. ロードマップの **どれから着手するか** を私に1問だけ尋ねる（私の想定は「2: Web Push バックエンド」か「3: ライブ連携プロキシ」）。
4. 着手前に、変更がデザイントークンと正確なデータ規約（5節）を壊さないことを確認する。

不明点は推測で埋めず、必ず先に質問してください。
